# NE301 Model Converter - CI/CD 配置实施计划

## Context（背景）

**问题**：项目当前没有任何 CI/CD 配置，导致：
1. 代码质量无法自动保障（无格式化、类型检查、代码检查工具）
2. 测试需要手动运行，容易遗漏
3. Docker 构建可能失败，但无法提前发现
4. 用户拿到代码后可能遇到部署问题

**目标**：配置完整的 CI 流程，确保：
- ✅ 代码质量一致性（自动格式化、类型检查、代码检查）
- ✅ 测试自动化（单元测试、集成测试、E2E 测试）
- ✅ Docker 构建验证（确保能成功构建和启动）
- ✅ 文档质量检查（Markdown 格式、链接有效性）
- ❌ 不需要自动部署到外部服务器

**预期效果**：
- 每次提交自动运行质量检查和测试
- PR 必须通过所有检查才能合并
- 确保 Docker 构建和启动一定成功
- CI 运行时间：10-15 分钟

### 阶段 3: Pre-commit Hooks 配置

**创建 `.pre-commit-config.yaml`**：

```yaml
repos:
  # Python 代码格式化
  - repo: https://github.com/psf/black
    rev: 24.8.0
    hooks:
      - id: black
        language_version: python3.11

  # Python 导入排序
  - repo: https://github.com/pycqa/isort
    rev: 5.13.2
    hooks:
      - id: isort
        args: ["--profile", "black"]

  # Python 代码检查
  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.6.4
    hooks:
      - id: ruff
        args: [--fix, --exit-non-zero-on-fix]

  # TypeScript/JavaScript 格式化
  - repo: https://github.com/pre-commit/mirrors-prettier
    rev: v3.3.0
    hooks:
      - id: prettier
        types_or: [typescript, javascript, css]
        files: ^frontend/

  # 通用检查
  - repo: https://github.com/pre-commit/pre-commit-hooks
    rev: v4.6.0
    hooks:
      - id: trailing-whitespace
      - id: end-of-file-fixer
      - id: check-yaml
        args: [--unsafe]
      - id: check-toml
      - id: check-json
      - id: check-added-large-files
        args: ['--maxkb=1000']
      - id: check-merge-conflict
```

### 阶段 4: GitHub Actions Workflow 配置

**创建 `.github/workflows/ci.yml`**：

```yaml
name: CI/CD Pipeline

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
  workflow_dispatch:

env:
  PYTHON_VERSION: "3.11"

jobs:
  # ==================== 代码质量检查 ====================
  lint-backend:
    name: Python 代码检查
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: ${{ env.PYTHON_VERSION }}
          cache: 'pip'

      - name: Install dependencies
        run: |
          python -m pip install --upgrade pip
          pip install -r backend/requirements.txt
          pip install -r backend/requirements-dev.txt

      - name: Run Black
        run: black --check backend/app backend/tests

      - name: Run isort
        run: isort --check-only backend/app backend/tests

      - name: Run Ruff
        run: ruff check backend/app backend/tests

      - name: Run MyPy
        run: mypy backend/app --ignore-missing-imports

  lint-frontend:
    name: TypeScript 代码检查
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
          cache-dependency-path: frontend/package-lock.json

      - name: Install dependencies
        run: |
          cd frontend
          npm ci

      - name: Run TypeScript type check
        run: |
          cd frontend
          npm run type-check

      - name: Run ESLint
        run: |
          cd frontend
          npm run lint

      - name: Run Prettier check
        run: |
          cd frontend
          npm run format:check

  # ==================== 单元测试 ====================
  test-backend-unit:
    name: 后端单元测试 (Python ${{ matrix.python-version }})
    runs-on: ubuntu-latest
    strategy:
      matrix:
        python-version: ['3.11', '3.12']

    steps:
      - uses: actions/checkout@v4

      - name: Set up Python ${{ matrix.python-version }}
        uses: actions/setup-python@v5
        with:
          python-version: ${{ matrix.python-version }}
          cache: 'pip'

      - name: Install dependencies
        run: |
          python -m pip install --upgrade pip
          pip install -r backend/requirements.txt
          pip install -r backend/requirements-dev.txt

      - name: Run unit tests
        run: |
          cd backend
          pytest tests/ -m unit -v --cov=app --cov-report=xml --cov-report=term-missing

      - name: Upload coverage report
        uses: actions/upload-artifact@v4
        if: matrix.python-version == '3.11'
        with:
          name: coverage-report
          path: backend/coverage.xml

  test-frontend-build:
    name: 前端构建测试
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
          cache-dependency-path: frontend/package-lock.json

      - name: Install dependencies
        run: |
          cd frontend
          npm ci

      - name: Build frontend
        run: |
          cd frontend
          npm run build

      - name: Upload build artifacts
        uses: actions/upload-artifact@v4
        with:
          name: frontend-build
          path: frontend/dist/

  # ==================== 集成测试 ====================
  test-integration:
    name: 后端集成测试
    runs-on: ubuntu-latest
    needs: [lint-backend, lint-frontend]

    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: ${{ env.PYTHON_VERSION }}
          cache: 'pip'

      - name: Install dependencies
        run: |
          python -m pip install --upgrade pip
          pip install -r backend/requirements.txt
          pip install -r backend/requirements-dev.txt

      - name: Run integration tests
        run: |
          cd backend
          pytest tests/ -m integration -v

  # ==================== E2E 测试 ====================
  test-e2e:
    name: E2E 测试
    runs-on: ubuntu-latest
    needs: [test-backend-unit, test-frontend-build]
    if: github.ref == 'refs/heads/main'

    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: ${{ env.PYTHON_VERSION }}
          cache: 'pip'

      - name: Set up Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
          cache-dependency-path: frontend/package-lock.json

      - name: Install backend dependencies
        run: |
          python -m pip install --upgrade pip
          pip install -r backend/requirements.txt
          pip install -r backend/requirements-dev.txt

      - name: Install frontend dependencies
        run: |
          cd frontend
          npm ci

      - name: Install Playwright browsers
        run: |
          npx playwright install --with-deps chromium

      - name: Pull NE301 Docker image
        run: docker pull camthink/ne301-dev:latest

      - name: Build Docker image
        run: docker-compose build

      - name: Start services
        run: docker-compose up -d

      - name: Wait for service ready
        run: |
          echo "等待服务启动..."
          sleep 10
          curl --retry 10 --retry-delay 5 --retry-connrefused http://localhost:8000/health

      - name: Run E2E tests
        run: |
          npx playwright test

      - name: Upload test results
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: playwright-report
          path: playwright-report/

      - name: Stop services
        if: always()
        run: docker-compose down -v

  # ==================== Docker 构建验证 ====================
  docker-build-test:
    name: Docker 构建验证
    runs-on: ubuntu-latest
    needs: [lint-backend, lint-frontend]

    steps:
      - uses: actions/checkout@v4

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Cache Docker layers
        uses: actions/cache@v4
        with:
          path: /tmp/.buildx-cache
          key: ${{ runner.os }}-buildx-${{ github.sha }}
          restore-keys: |
            ${{ runner.os }}-buildx-

      - name: Pull NE301 base image
        run: docker pull camthink/ne301-dev:latest

      - name: Build Docker image (no push)
        uses: docker/build-push-action@v5
        with:
          context: .
          file: ./backend/Dockerfile
          push: false
          cache-from: type=local,src=/tmp/.buildx-cache
          cache-to: type=local,dest=/tmp/.buildx-cache-new,mode=max

      - name: Move Docker cache
        run: |
          rm -rf /tmp/.buildx-cache
          mv /tmp/.buildx-cache-new /tmp/.buildx-cache

  # ==================== Docker 启动验证 ====================
  docker-compose-test:
    name: Docker Compose 启动验证
    runs-on: ubuntu-latest
    needs: [docker-build-test]

    steps:
      - uses: actions/checkout@v4

      - name: Pull NE301 base image
        run: docker pull camthink/ne301-dev:latest

      - name: Build and start services
        run: docker-compose up -d --build

      - name: Wait for service ready
        run: |
          echo "等待服务启动..."
          sleep 15
          for i in {1..30}; do
            if curl -f http://localhost:8000/health; then
              echo "✅ 服务健康检查通过"
              exit 0
            fi
            echo "等待服务启动... ($i/30)"
            sleep 2
          done
          echo "❌ 服务健康检查失败"
          docker-compose logs
          exit 1

      - name: Check frontend accessibility
        run: |
          curl -f http://localhost:8000/ || exit 1
          echo "✅ 前端访问正常"

      - name: Check API accessibility
        run: |
          curl -f http://localhost:8000/api/setup/check || exit 1
          echo "✅ API 访问正常"

      - name: Show logs on failure
        if: failure()
        run: docker-compose logs

      - name: Stop services
        if: always()
        run: docker-compose down -v

  # ==================== 文档检查 ====================
  docs-check:
    name: 文档质量检查
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Check Markdown files exist
        run: |
          test -f README.md && echo "✅ README.md exists"
          test -f CLAUDE.md && echo "✅ CLAUDE.md exists"
          test -f README.docker.md && echo "✅ README.docker.md exists"

      - name: Validate markdown format
        uses: DavidAnson/markdownlint-cli2-action@v16
        with:
          globs: |
            *.md
            docs/**/*.md
          config: |
            {
              "MD013": false,
              "MD033": false,
              "MD041": false
            }
```

## 注意事项

1. **首次运行**：首次运行 pre-commit 可能会格式化大量文件，建议单独提交格式化更改

2. **依赖安装**：运行 `./scripts/install-dev-tools.sh` 安装所有开发工具

3. **CI 缓存**：GitHub Actions 会缓存 pip 和 npm 依赖，第二次运行会更快

4. **Docker 镜像**：CI 中只构建镜像不推送，确保构建成功即可

5. **E2E 测试**：只在 main 分支运行，节省 PR 验证时间

6. **测试标记**：确保测试正确使用 `@pytest.mark.unit` 和 `@pytest.mark.integration` 标记
