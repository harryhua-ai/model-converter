## 🎯 产品定位

**个人开发工具** - 用于将 PyTorch 模型转换为 NE301 设备可用的 .bin 文件

### 核心价值主张

- **零代码操作**: 无需理解底层转换流程，界面化操作
- **端到端自动化**: PyTorch → 量化 → NE301 .bin 全自动

- **实时反馈**: WebSocket 推送转换进度，不阻塞界面
- **跨平台兼容**: 支持 macOS / Linux / Windows（通过浏览器）

## 💡 类别定义 YAML 说明

### 为什么需要类别定义文件？

在目标检测模型中，模型输出的每个检测框需要映射到具体的类别。类别定义文件（YAML）用于：

1. **后处理**: 将模型输出映射到人类可读的类别名称
2. **可视化**: 在结果中显示正确的类别标签和颜色
3. **验证**: 确保类别数量与模型配置一致

### YAML 文件格式

```yaml
# classes.yaml - 类别定义模板

# 方式 1: 简单列表（适合新手）
classes:
  - person
  - car
  - dog

# 方式 2: 完整定义（推荐）
classes:
  - name: person
    id: 0
    color: [255, 0, 0]      # RGB 颜色，用于可视化

  - name: car
    id: 1
    color: [0, 255, 0]

  - name: dog
    id: 2
    color: [0, 0, 255]

# 方式 3: 包含更多元数据
classes:
  - name: person
    id: 0
    display_name: 人  # 中文显示名
    color: [255, 0, 0]

  - name: car
    id: 1
    display_name: 汽车
    color: [0, 255, 0]
```

### 用户引导

对于不熟悉 YAML 的用户，提供：

1. **模板下载**: 一键下载标准模板文件
2. **在线编辑器**: 简单的表单界面，自动生成 YAML
3. **实时验证**: 上传时自动检查格式
4. **示例预览**: 显示常见配置（COCO 80 类、自定义等）

## 🎨 核心功能

### MVP 功能集（Phase 1）

#### 1. 环境检测和设置 ✅
- 自动检测 Docker 环境
- Docker 安装引导（针对不同操作系统）
- 镜像状态检查
- 一键安装脚本

#### 2. 模型上传 ✅
- 支持格式：`.pt`, `.pth`, `.onnx`
- 文件大小限制：500MB
- 拖拽上传 + 点击选择
- 自动文件验证

#### 3. 类别定义文件上传（可选）✅
**用途**: 定义模型检测的类别信息，用于后处理和结果解析

**格式**: YAML 文件

**示例**:
```yaml
# classes.yaml
classes:
  - name: person
    id: 0
    color: [255, 0, 0]
  - name: car
    id: 1
    color: [0, 255, 0]
  - name: dog
    id: 2
    color: [0, 0, 255]
```

**功能**:
- 支持 `.yaml` / `.yml` 格式
- 文件大小限制：10KB
- 提供在线模板下载
- 实时格式验证
- 支持手动编辑和预览
- 显示类别数量和名称列表

#### 4. 配置预设 ✅
**3 个快速预设**：
- **快速模式**: 256x256, INT8 - 速度优先
- **平衡模式**: 480x480, INT8 - 推荐
- **高精度模式**: 640x640, INT8 - 精度优先

#### 6. 参数自定义 ✅
**可调参数**：
- 模型类型：YOLOv8 / YOLOX
- 输入尺寸：256/480/640（可选）
- 类别数量：1-1000
- 置信度阈值：0.01-0.99
- 量化类型：INT8（固定）

#### 7. 校准数据集（可选）✅
- 支持 `.zip` 格式
- 建议：32-100 张代表性图片
- 文件大小限制：1GB
- 自动解压和验证

#### 8. 实时进度显示 ✅
- WebSocket 推送（非轮询）
- 进度百分比（0-100%）
- 当前步骤描述
- 预计剩余时间
- 可随时取消

#### 9. 结果下载 ✅
- 转换完成后自动提示
- 文件名包含版本号和时间戳
- 下载前显示文件信息（大小、配置）

### 明确不做（Phase 1）

- ❌ 批量转换（留待 Phase 2）
- ❌ 任务历史记录
- ❌ 用户认证系统
- ❌ 多任务并发处理
- ❌ 远程访问支持
- ❌ 本地工具链支持（仅 Docker）
- ❌ 高级量化参数配置（INT8 固定）
- ❌ 模型结构分析和可视化

## ⚠️ 风险和限制

### 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| Docker 依赖 | 高 | 智能检测 + 一键安装脚本 + 友好引导 |
| 镜像拉取时间 | 中 | 实时进度推送 + 后台预加载 + 国内镜像加速 |
| 容器启动失败 | 低 | 健壮的错误处理 + 友好错误提示 |
| 网络问题 | 低 | 支持本地镜像文件 + 离线安装选项 |

### 产品限制

- 仅支持 YOLOv8 / YOLOX 模型
- 仅支持 INT8 量化
- 单任务处理
- 本机使用（不支持远程）
- 需要安装 Docker Desktop
